# Game Feel for Code-Built UI

코드로 UI를 그리는(씬 대신 C#에서 Canvas/Image/Text 생성) 게임에서 "데모앱" 느낌을 지우고 게임처럼 보이게 다듬은 실전 노트. 플레이 테스트에서 반복해서 지적된 것들을 정리했습니다.

## 데모처럼 보이는 신호들 (그리고 해법)

- **각진 네모.** 기본 사각 패널·버튼은 폼(form)처럼 보입니다. 런타임에 둥근 사각 스프라이트(Texture2D로 rounded-rect 생성 + `Image.Type.Sliced` 9-slice)를 만들어 패널·버튼에 씌우면 부드러워집니다.
- **개발자 메트릭 라벨.** "N hits / M before", "clear 3.2s" 같은 디버그성 텍스트가 화면에 있으면 미완성처럼 보입니다. 플레이어 화면에서는 **숨기되**, 검증(self-check)이 읽어야 하면 텍스트 값은 계속 채우고 `SetActive(false)`로 표시만 끕니다.
- **기계적인 명명.** "Slime Grove / Wave 24" 같은 이름은 플레이스홀더처럼 읽힙니다. 세계관에 맞게 테마화하세요(지역명 + "Depth N"처럼 진행이 세계관 안에서 의미 있게).
- **강조색 CTA, 대중없는 텍스트 위치.** 주황색 버튼 배경이 게임 톤과 안 맞으면 데모 느낌이 납니다. 색·위치를 한 기준선(anchor)에 맞춰 정렬하고, 자동 진행 게임이면 불필요한 CTA는 아예 숨깁니다.

## 이펙트는 뜨고 사라져야 자연스럽다

- **슬래시·플래시는 자동 페이드.** 히트 이펙트가 화면에 계속 남아 있으면 지저분합니다. 짧게 보여준 뒤(예: 0.16s) 코루틴으로 알파를 0까지 내려 사라지게 합니다.
- **데미지 숫자는 팝 + 소멸.** 데미지 텍스트가 대상 위로 살짝 떠오르며 페이드아웃하면 타격감이 삽니다. 정지된 큰 숫자는 UI처럼 보입니다.
- **동적 효과는 스크린샷에 안 보인다.** 페이드·팝·흔들림은 정지 캡처로 확인되지 않으니, 애니 확인은 `recordVideo`나 실기로 합니다(→ `04-app-store-connect-testflight/VisualVerification.md`).

## 시뮬레이터 스크린샷을 볼 때 주의

- **상단 중앙의 검은 알약(pill)은 게임 UI가 아닙니다.** iPhone 14 Pro 이후 기기·시뮬레이터의 **Dynamic Island**가 캡처에 찍힌 것입니다. 게임 요소로 오인해 "지우려" 하지 마세요 — 실기에서도 항상 그 자리에 있습니다.
- 노치·Dynamic Island·상태바 영역을 피해 상단 HUD를 배치했는지 확인하세요.

## 코드 빌드 버튼의 탭 함정 (실기에서만 드러난 버그)

코드로 버튼을 만들 때 **라벨(Text)을 버튼의 자식이 아니라 형제로 겹쳐 올리면, 라벨이 탭을 삼켜 버튼이 안 눌립니다.** Unity UI 레이캐스트는 최상단 그래픽에 떨어지는데, Text의 `raycastTarget` 기본값이 true라 "글자를 정확히 누른" 탭이 버튼에 전달되지 않습니다 — 사용자는 "버튼이 안 먹는다"고 느낍니다.

- **시뮬레이터 스크린샷으로는 절대 안 잡힙니다.** 화면은 완벽해 보이고, 실제 터치가 있어야만 드러납니다. 탭 인터랙션은 실기 검증 항목으로 분류하세요.
- **해법: 클릭 대상이 아닌 모든 그래픽은 `raycastTarget = false`.** 라벨 생성 헬퍼(CreateText류)에 기본으로 박아두는 게 가장 안전합니다. 버튼 위 아이콘·장식 패널·전체화면 연출(플래시 등)도 마찬가지입니다.
- **전체화면 탭 영역(컷신 "아무 데나 탭") 위에 다른 버튼이 뜨는 상태**(선택지 등)에서는, 배경 탭 핸들러에 상태 가드를 두세요 — 안 그러면 선택지 옆 빈 곳을 탭했을 때 배경 핸들러가 장면을 넘겨버립니다.
