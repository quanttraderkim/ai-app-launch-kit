# PRD: Streak

> 예시 문서입니다. `02-product-prd/PRD.template.md`를 채운 것이며, 계정·서버 값은 없습니다(로컬 저장만).

## Product Vision

매일 작은 습관 하나(물 마시기, 스트레칭 등)를 지키고 싶지만 복잡한 트래커 앱은 부담스러운 사람이, 하루 한 번 버튼을 눌러 "오늘도 지켰다"를 기록하고 연속 기록이 쌓이는 걸 보며 동기를 얻는다.

## Core Value Sentence

오늘 한 번의 탭으로 "지켰다"를 남기고, 연속 숫자가 +1 올라가는 걸 보는 순간.

## Target User

하루 1분도 앱 조작에 쓰기 싫은, 습관 하나를 꾸준히 이어가려는 사람. 통계나 설정보다 "이어가는 느낌"이 중요한 사람.

## First Usable Proof

앱을 연다 → 큰 "오늘 완료" 버튼을 한 번 탭한다 → streak이 0에서 1로 바뀐다 → 앱을 껐다 다시 열어도 1이 유지된다. 계정·설정·네트워크 없이 동작해야 한다.

## MVP Scope

- 습관 하나, 하루 1회 체크
- 연속 일수(streak) 카운트와 큰 숫자 표시
- 로컬 저장(껐다 켜도 유지)

## Explicitly Out Of Scope

- 여러 습관 관리
- 알림/리마인더
- 계정·서버·동기화
- 통계 그래프·달력 뷰

## Core Loop

```text
앱 열기 -> "오늘 완료" 탭 -> streak +1, 오늘은 완료 표시 -> 내일 다시 열어 이어가기
```

## UX Quality Bar

- 핵심 행동은 1탭. 홈 화면에서 바로 "오늘 완료"가 보인다.
- streak 숫자는 화면에서 가장 크게, 즉시 읽힌다.
- 오늘 이미 완료했으면 버튼이 "오늘 완료됨" 상태로 바뀌어 중복 탭을 막는다.

## Data And Privacy

첫 proof는 로컬 저장(UserDefaults)만 사용. 계정·서버·개인정보 수집 없음. 저장값은 마지막 완료 날짜와 현재 streak 두 개뿐.

## Release Path

TestFlight 내부 테스트 그룹(친한 사람 몇 명)로 첫 proof를 보여주고 반응을 관찰한다.

## Domain Glossary

| Term | Definition |
| --- | --- |
| Streak | 하루도 빠지지 않고 이어진 완료 일수 |
| Check | 특정 하루의 "오늘 완료" 기록 (하루 최대 1회) |

## Learning Questions

1. 사용자가 "오늘 완료" 버튼의 의미를 바로 이해했는가?
2. streak 숫자가 올라가는 것을 가치 있게 느꼈는가?
3. 다음 날 다시 열었는가, 아니면 한 번 쓰고 말았는가?
